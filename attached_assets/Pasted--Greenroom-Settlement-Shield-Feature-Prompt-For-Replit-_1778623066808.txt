## 2. Feature overview: Settlement Shield

Two linked sub-features shipped together.

### 2a. Smart Switch
At the moment a deal is entered or edited, Greenroom analyses the deal type and expected gross, generates a **flat guarantee equivalent**, and offers to lock the deal as a flat. This eliminates settlement complexity and dispute risk before show night.

Two flavours:

| Trigger | What happens |
|---------|-------------|
| Deal type is `vs`, `percentage_of_net`, or `flat` AND expected gross < $5,000 | Suggest converting to flat at the calculated equivalent |
| Deal type is `door` (any gross) | Suggest converting to the door hybrid structure (floor + split) |

### 2b. Insurance
Two opt-in products, offered at the moment the switch is accepted. Either party (venue or artist) can enroll one-sided, or both can enroll two-sided (shared cost, 15% cheaper each).

| Product | Trigger | Pays |
|---------|---------|------|
| **P1 — Dispute instant pay** | Settlement enters `disputed` status | Resolution amount, up to tier cap, within 24h |
| **P2 — Suggestion accuracy pay** | Actual payout differs from flat suggestion by > $50 | The gap, up to X = $400 (99.2% coverage) |

## 3. Precise calculations

### 3a. Smart Switch — under $5K flat suggestion

> Key finding from 24 months of data: **100% of vs deals under $5K paid the guarantee** — the percentage calculation never fired once. The vs deal is functionally a flat deal at the guarantee amount. The suggestion is therefore exact, not estimated.

**For `vs` deals (expected gross < $5,000):**
```
flat_suggestion = guarantee_amount
```
Accuracy: avg delta $11 (rounding only). No history needed.

**For `percentage_of_net` deals (expected gross < $5,000):**
```
venue_avg_expenses = 1497          // from 24-month historical average
capped_exp = min(venue_avg_expenses, expense_cap ?? venue_avg_expenses)
flat_suggestion = percentage × (expected_gross × 0.9 − capped_exp)
flat_suggestion = round(flat_suggestion / 50) × 50   // round to nearest $50
```
Accuracy: avg delta $24, P90 delta $50.

**For `flat` deals:** Already flat — no switch needed. Show a "this deal is already clean" indicator instead.

**Confidence tiers** (affects display band and default insurance tier):
| Tier | Condition | Display | Default insurance |
|------|-----------|---------|------------------|
| A | Artist has 2+ shows on platform | Single number `$X` | Tier 2 |
| B | Artist has 1 show OR agent has history | `$X` with ± $50 note | Tier 2 |
| C | New artist, agent has platform history | `$X – $Y` range | Tier 3 |
| D | Cold booking — no artist or agent history | Range + low-confidence flag | Tier 4 |

---

### 3b. Smart Switch — door deal hybrid

> Key finding: Current door deals give the venue $0 — they absorb all fixed costs and hand the remainder to the artist. The hybrid gives both parties a genuine share.

**Formula (applies to door deals at any gross level):**
```
ticketing_fees = gross × 0.10
exp_input      = min(actual_expenses, 1500)   // cap at venue historical avg
avail          = gross × 0.90 − exp_input     // available pool

if avail ≤ 500:
  artist = 500                                 // floor guarantee
  venue  = avail − 500                         // negative; venue absorbs

if avail > 500:
  above  = avail − 500
  artist = 500 + (0.60 × above)               // 60% of pool above floor
  venue  = (0.40 × above) − (actual_expenses − exp_input)  // 40% minus any overage
```

**Insurance gap on door deals**: when `actual_expenses > exp_input`, the overage is the venue's responsibility. Insurance (P2, Tier 4+) covers this gap up to X = $400.

**Settlement display formula** (show this to both parties):
```
Artist = $500 floor + 60% of available pool above $500
Venue  = 40% of available pool above $500 (expenses already deducted)
```

## 5. When to introduce the feature in the deal workflow

The suggestion must fire **at deal entry / deal edit** — not at settlement. By settlement night it is too late. The goal is to resolve the deal structure weeks before show night.

### Exact trigger points:

1. **On `POST /api/deals` or deal edit save**: After the deal record is saved, immediately call `generateSwitchSuggestion(deal, show)`. If the result is non-null, save a `switch_suggestions` record and set `deal.switchStatus = 'suggested'`.

2. **In the show detail page (`app/shows/[id]/page.tsx`)**: Show the suggestion card prominently in the Deal section if `deal.switchStatus === 'suggested'`. Include a "Suggest to agent" button that would send (or copy) the structured proposal.

3. **T-7 advance reminder**: When show status transitions from `booked` → `advanced` and the switch hasn't been accepted, surface a reminder banner.

4. **At settlement entry (`app/shows/[id]/settle/page.tsx`)**: If the switch was accepted, pre-fill the settlement with the flat amount and show a one-click confirm. If it was declined or not yet addressed, show the original deal and the suggestion as an inline nudge.

### State machine for the switch:
```
not_suggested
  → suggested (deal saved, system generated a suggestion)
    → accepted (Mariana or agent accepted)
      → insurance_offered (show insurance enrollment UI)
        → insured / not_insured
    → declined (agent declined)
      → original deal stands
```

## 9. Key constraints to respect

1. **Do not break the existing `calculateSettlement` function.** It handles `flat` and `percentage_of_gross` correctly. Add switch logic in the new `switchMath.ts` and extend the settlement page separately.

2. **The switch is always opt-in.** Never auto-convert a deal. The system suggests; Mariana and the agent decide.

3. **The suggestion fires at deal entry, not at settlement.** Add the trigger in the deal save API route. Do not surface it only in the settlement view — that's too late.

4. **Confidence tier D**: For completely unknown artists (no platform data), always show the suggestion as a range and flag it. Do not auto-send to agent — surface to Mariana first for manual review.

5. **The $5K gross boundary is hard for flat suggestions.** Do not suggest flat conversion for vs or percentage_of_net deals where expected gross ≥ $5,000. At that level the percentage calculation genuinely fires and a flat would shortchange artists. The boundary may be shown as a tooltip: "At this expected gross level, switching to flat could reduce artist pay. Switch not recommended."

6. **Door hybrid applies at any gross level.** There is no upper bound for the door hybrid — even large-gross door shows (Pale Lake at $19K) benefit from restructuring.

7. **The insurance is priced per-show at booking.** It is not a monthly subscription. The premium is added to the venue's Greenroom invoice for each show where insurance was enrolled.
